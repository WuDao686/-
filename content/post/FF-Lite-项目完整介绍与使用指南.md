# FF-Lite 仿飞书协同办公平台 - 完整项目介绍与使用指南

---

## 一、项目概述

### 1.1 一句话定位

**为中小企业提供 2 周即可上线的协同办公 MVP**，覆盖"找人、聊天、写文档、管账号、管组织、管任务"六大场景，让 50 人以下团队无需采购 SaaS 也能跑通数字化办公闭环。

### 1.2 产品代号

**FF-Lite MVP**（仿飞书协同办公平台最小可行产品）

### 1.3 核心价值

| 痛点         | 传统做法              | FF-Lite 解法               |
| :----------- | :-------------------- | :------------------------- |
| 团队沟通分散 | 微信 + 邮件混用       | 工作消息全在一处，实时推送 |
| 找人困难     | 翻微信群 / 问同事     | 通讯录检索 + 员工名片      |
| 文档协作混乱 | 邮件附件 + 微信传文件 | 本地编辑 + 群组分享        |
| 账号管理乱   | 共享账号 / 密码群发   | 用户管理 + RBAC + 操作日志 |
| 组织架构不清 | 群聊混乱              | 部门树 + 职位管理          |
| 任务追踪困难 | 口头/微信派活         | 任务发布/分配/追踪/日程    |

### 1.4 项目状态

| 版本  | 范围                                                      | 功能数 |  状态  |
| :---: | :-------------------------------------------------------- | :----: | :----: |
| V1.0  | 用户中心 / 通讯录 / IM / 文档 / 管理后台                  |   22   | 已完成 |
| V2.0  | 多企业 SaaS 化（企业注册 / 邀请码 / 删除企业 / 数据隔离） |   4    | 已完成 |
| V2.1  | 任务与日程管理                                            |   5    | 已完成 |
| V2.1+ | AI 智能助理                                               |   5    | 规划中 |

## 二、技术架构

### 2.1 技术栈一览

| 层级          | 技术选型                  | 说明                              |
| :------------ | :------------------------ | :-------------------------------- |
| **前端**      | Vue 3 + TypeScript + Vite | SPA 单页应用，Element Plus 组件库 |
| **后端**      | .NET 10 + C#              | 模块化单体架构，DDD 领域驱动设计  |
| **数据库**    | PostgreSQL 16+            | 多 Schema 隔离，EF Core ORM       |
| **缓存/消息** | Redis 7+                  | 缓存 + SignalR Backplane 实时推送 |
| **文件存储**  | MinIO                     | 对象存储（图片/文件上传）         |
| **实时通信**  | SignalR                   | WebSocket 实时消息推送            |
| **认证**      | JWT (HS256)               | 无状态 Token 认证                 |
| **部署**      | Docker Compose            | 一键启动 5 个容器                 |
| **反向代理**  | Nginx + Let's Encrypt     | HTTPS 加密 + 域名转发             |

## 四、角色权限体系

系统预置 3 个固定角色，不可自定义：

| 角色           | 权限范围                            |
| :------------- | :---------------------------------- |
| **超级管理员** | 所有权限，含管理后台全部功能        |
| **部门主管**   | 查看本部门成员，发布/管理本部门任务 |
| **普通员工**   | 找人、聊天、写文档、查看自己的任务  |

## 五、多企业 SaaS 隔离

V2.0 实现了多企业数据隔离，核心机制：

1. **共享数据库 + `tenant_id` 软隔离**：所有业务表加 `tenant_id` 列
2. **EF Core 拦截器**：自动注入 `tenant_id`，开发无需手动填写
3. **QueryFilter**：查询自动过滤本企业数据
4. **跨企业禁通讯**：私聊/群聊/文档分享/通讯录查询均限制在同企业内
5. **1 手机号 = 1 企业**：不支持多企业身份切换
6. **企业邀请码**：格式 `COMP-XXXXXX-XXXX`，超管可重置 Salt 使旧码失效

## 七、快速使用指南

### 7.1 登录

1. 打开浏览器访问 `https://fs.lctnb.top`
2. 输入演示账号手机号和密码（如 `13999990001` / `Test@1234`）
3. 点击"登录"按钮进入工作台

### 7.2 即时通讯

1. 点击左侧菜单"即时通讯"
2. 在通讯录中找到同事 → 点击"发起聊天"
3. 输入文字消息发送，支持图片和文件上传
4. 群聊在会话列表中显示，点击进入即可

### 7.3 任务管理

1. 点击左侧菜单"我的任务"查看分配给你的任务
2. 超管/部门主管可点击"发布任务"创建新任务
3. 任务详情中可标记完成、查看进度
4. 点击"日程安排"以日历视图查看任务
5. 点击"任务进度"查看发布任务的完成统计

### 7.4 云文档

1. 点击左侧菜单"云文档"进入文档列表
2. 点击"新建文档"打开富文本编辑器
3. 编辑完成后点击"保存"
4. 文档详情中可分享到群组

### 7.5 管理后台

1. 超管在侧边栏底部点击"管理后台"
2. **企业概览**：查看企业信息、邀请码
3. **用户管理**：新增员工（生成邀请链接）、编辑/禁用/重置密码
4. **部门管理**：新增/编辑/删除部门
5. **职位管理**：新增/编辑/删除职位
6. **操作日志**：查看所有关键操作记录

### 7.6 注册新企业

1. 在登录页点击"注册企业"
2. 填写手机号、企业名称、密码
3. 注册成功后自动成为该企业的超级管理员
4. 在管理后台可查看企业邀请码，用于邀请员工加入

3. **全功能测试**：验证了登录、注册、用户信息获取、任务列表、即时通讯、通讯录、云文档、管理后台等所有核心功能均正常运行

### 技术要点

- JWT HS256 算法要求密钥至少 **128 位（16 字符）**，生产环境建议使用 32 字符以上随机串
- 模块化单体架构通过 DDD 限界上下文 + 多 Schema 数据库实现模块隔离
- 多企业 SaaS 通过共享 DB + `tenant_id` 软隔离 + EF Core 拦截器实现，无需拆微服务
